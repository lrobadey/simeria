Adapa reasoning redesign

From ladder to living loop

Replace the one-way survival script with a recurring appraisal cycle where needs, memory, risk, and opportunity compete every day.

The pivot

The current chain asks what branch comes next. The new loop asks what pressure matters most now.

Current ladder

  1. Eat if hungryBody need interrupts everything else.
  2. Sleep if nightEnergy and darkness trigger a fixed rest branch.
  3. Forage if pack is lowFood stock is a simple threshold.
  4. Build shelterThe house project is inevitable once food is stable.
  5. Stockpile and idleHome life is two thresholds: food and wood.

Target loop

SenseRead hunger, energy, daylight, weather, animals, water, known resources.
RememberUpdate places, failures, prey ranges, scarcity, danger, reliable food.
AppraiseTurn state into live pressures and opportunity scores.
CompeteScore all viable projects against the current active project.
ActDo the next small action for the chosen project.
LearnFeed the cost, result, and discovery back into memory.

Pressure model

Pressures are not tasks. They are reasons. Tasks become realistic when they answer several reasons at once.

HungerBody hunger, stored food, known food reliability, season.
FatigueEnergy, daylight remaining, distance from safe rest.
ExposureBad sleep, floodwater, night risk, lack of roof or fire.
ScarcityLow known reeds, wood, food, or repeated failed search.
RiskDarkness, deep water, predator proximity, boar danger, injury.
OpportunityRipe dates, visible prey, fish flood pools, fresh deadwood.
CapabilityTools, materials, memory quality, recent success or failure.
CuriosityUnknown nearby map, stale memory, blocked project alternatives.

project score = pressure answered + opportunity + readiness - cost - risk

Project candidates

The active project continues only while it remains competitive. Shelter can pause; hunger can subside; hunting can wait for a better opening.

ForageBest when hunger is high, safe food is remembered, or daylight is short.
RestBest when fatigue or darkness makes other work costly.
Build shelterBest when exposure is high, materials are known, and food pressure is low.
Gather fuelBest when night approaches, hearth value is high, or deadwood is nearby.
ExploreBest when memory is stale, resources are blocked, or local scarcity rises.
Make toolBest when a tool unlocks several blocked projects, especially hunting or fishing.
Hunt or fishBest when food pressure and prey opportunity beat risk and energy cost.
Relocate campBest when current ground repeatedly fails: flood, food, fuel, safety.

Implementation path

Refactor in layers so each step preserves the current behavior while opening space for more realistic choices.

1. Extract pressuresAdd `assessPressures()` from hunger, energy, daylight, memory, flood, and inventory.
2. Generate projectsReplace the ladder body with candidate builders for forage, rest, shelter, fuel, explore.
3. Score and persistKeep `activeProject` if it still scores near the winner; otherwise switch with a memory reason.
4. Add capabilityIntroduce crude tools and make hunting/fishing depend on method, terrain, and opportunity.
5. Learn from outcomesEvery success, failure, injury, blocked route, or rich patch changes future scores.
